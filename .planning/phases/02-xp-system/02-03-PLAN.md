  <action>
    Add the XP bar JSX block to PartyTab in app/game/[id]/page.tsx:

    INSERTION POINT: immediately after the closing </div> of the HP bar block (after line 574 in the
    current file), before the opening of the compact 6-stat row div.

    The XP bar uses an IIFE (immediately invoked function expression) to keep the computation local
    to each member in the .map() callback. This is the same pattern used in page.tsx for complex
    per-member rendering. The exact computation logic:

    - xp = m.character.xp (the character's total cumulative XP)
    - level = m.character.level (1–5)
    - atCap = level >= 5
    - nextXp = atCap ? null : xpForNextLevel(level) (the cumulative XP threshold for the next level)
    - prevXp = XP_THRESHOLDS[level - 1] (cumulative XP at the START of the current level)
    - xpInLevel = xp - prevXp (XP earned within the current level)
    - xpNeeded = nextXp !== null ? nextXp - prevXp : 1 (XP span of the current level)
    - xpPct = atCap ? 100 : Math.max(0, Math.min(100, (xpInLevel / xpNeeded) * 100))
    - label = atCap ? "Level 5  ·  MAX" : "Level " + level + "  ·  XP: " + xp + " / " + nextXp

    Render:
    - Outer div: className="space-y-1" (same as HP bar wrapper)
    - Label row: className="text-xs text-slate-500" with the label text (left-aligned only, no right column)
    - Track: className="h-1.5 w-full bg-slate-200 rounded-full overflow-hidden" (h-1.5 is slimmer than HP bar's h-2)
    - Fill: className="h-full rounded-full bg-blue-500" with style={{ width: xpPct + "%" }}

    After completing the JSX edit, start the dev server (npm run dev) to visually confirm the XP bar renders in the Party tab. Then pause for human verification.
