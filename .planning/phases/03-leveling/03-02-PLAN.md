## Phase Goal

**As a** player creating a new character, **I want to** start with the correct class-aware max HP (Fighter 10+CON, Rogue/Cleric 8+CON, Wizard 6+CON) that persists to every adventure I join, **so that** my Wizard isn't silently as tanky as a Fighter and my HP carries over correctly when the game initializes party state.

<objective>
Wire the new `maxHpAtLevel` helper and `Character.maxHp` column (built in Plan 01) into the two server actions that produce / consume the canonical value: `create-character.ts` writes the class-correct starting HP at character creation (D-02), and `start-adventure.ts` reads `character.maxHp` straight from the DB instead of recomputing with the hardcoded `10 + conMod` formula on line 49 (D-03).

Purpose: Replaces the long-standing `hp = 10 + Math.floor((member.character.constitution - 10) / 2)` magic number in start-adventure.ts (which gave every class identical HP) with the canonical persistent column. Establishes character creation as the single point where class-aware HP is computed for level 1.

Output: Two surgically edited server action files. No new files. No new dependencies.
</objective>
