## Phase Goal

**As a** D&D player whose character just defeated a tough encounter, **I want to** have my level, max HP, and proficiency bonus all updated atomically in the same database write as my XP grant — even when one big XP grant takes me up multiple levels in a single turn, **so that** my level-up never leaves the database in a half-updated state and I cannot be cheated out of HP by a multi-level skip.

<objective>
Extend the existing take-turn.ts $transaction (already wired in Phase 2 to update xp and level) so it ALSO writes the class-correct maxHp value alongside, and returns a structured levelUpResult payload that Plan 04 UI will consume. Verify that RULES_ENGINE_KEYS at line 299 already covers all four LVL-04 keys (hp, maxHp, xp, level) plus proficiencyBonus — the LVL-04 allowlist work is a no-op verification step (D-12). Add one multi-level-up unit test to lib/leveling.test.ts proving the total-HP function is correct for an XP jump from 0 to 6500 in a single turn.

Purpose: Closes the LVL-02 atomic level-up transaction (HP grows alongside level per D-04), exposes the levelUpResult server response shape (per D-07) that Plan 04 will render as a card, and documents LVL-04 as already-met (per D-12).

Output: One edited server action with four surgical changes (import, interface extension, $transaction extension, return-value extension) and one extra unit test added to lib/leveling.test.ts.
</objective>
