## Phase Goal

**As a** D&D player, **I want to** have my character's max HP automatically computed by class-correct hit-die math (Fighter d10, Rogue/Cleric d8, Wizard d6) and persisted to the database, **so that** when I level up my HP grows by the right amount and never gets silently clamped to a wrong default value.

<objective>
Ship the math + persistence foundation for Phase 3 as ONE vertical slice: add the `maxHp Int` column to `Character`, create the pure `lib/leveling.ts` module with `maxHpAtLevel` + `HIT_DIE_BY_CLASS`, write the LVL-05 unit tests, run the first-ever `prisma migrate dev`, then backfill every existing character row with its class-correct maxHp using `maxHpAtLevel(class, con, level)`.

Purpose: Establishes the single source of truth for HP math (per D-08/D-09/D-10) and the canonical persistent column (per D-01) that Plans 02 and 03 will consume. Replicates the proven `lib/xp.ts` pure-module pattern so tests run with zero DB or framework dependencies.

Output: A new pure module + test file, an extended Prisma schema with applied migration, and corrected `maxHp` values on every pre-existing character row.
</objective>
