## Phase Goal

**As a** D&D player, **I want to** choose my character's skill proficiencies from a class-specific list during character creation, **so that** my character has the correct skills persisted to the database and reflected in the game UI.

<objective>
Schema migration adds skillProficiencies String[] to the Character model; the character creation form grows a class-gated skill multi-select step; and a backfill script seeds thematic defaults for existing characters.

Purpose: Skill proficiencies are the foundational data that powers the Stats sub-tab proficiency highlight (04-05) and in-game skill check resolution (04-04). This plan establishes the DB column and the creation-time UI/validation layer.

Output: Migration SQL, regenerated Prisma client, seed-backfill-skills.mjs script, updated character-form.tsx with skill picker, updated create-character.ts with server-side validation, CharacterData interface extended in page.tsx.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client form → create-character.ts | Skill picks submitted as JSON string in FormData; client can be bypassed |
| create-character.ts → Prisma | Server action is the only write path to Character.skillProficiencies |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | create-character.ts skillProficiencies parse | mitigate | JSON.parse wrapped in try/catch; return { success: false } on invalid data; cannot crash server action |
| T-04-01-02 | Tampering | CLASS_SKILL_POOL allowed-list check | mitigate | Server-side: every(s => allowedSkills.includes(s)) before prisma write; UI bypass cannot insert out-of-class skills |
| T-04-01-03 | Tampering | SKILL_PICK_COUNT enforcement | mitigate | Server-side length check: skillProficiencies.length !== requiredCount returns error; UI cannot submit wrong count |
| T-04-01-04 | Tampering | Backfill script class override | accept | Script skips unknown classes with console.warn; no unauthenticated access path; runs once in developer terminal |
| T-04-01-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; all imports are from existing package.json dependencies |
</threat_model>
