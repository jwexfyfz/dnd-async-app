## Phase Goal

**As a** D&D player, **I want to** see my character's class features in the Abilities sub-tab populated from the database, **so that** newly unlocked features are flagged on level-up and the data is always current with the seeded ClassFeature table.

<objective>
Replace the hardcoded CLASS_FEATURES map in page.tsx with a live DB fetch via a new getClassFeatures server action. MemberAbilitiesPane uses useEffect to fetch features on mount and on level change, renders them grouped by level with a "New" badge on features at the current character level.

Purpose: The ClassFeature table is already seeded for Fighter/Rogue/Cleric/Wizard levels 1-20. Wiring the Abilities sub-tab to the DB eliminates the flat hardcoded list and enables the level-up visual indicator without any future code changes.

Output: app/actions/get-class-features.ts (new file), page.tsx with CLASS_FEATURES deleted and MemberAbilitiesPane rewritten.
</objective>
