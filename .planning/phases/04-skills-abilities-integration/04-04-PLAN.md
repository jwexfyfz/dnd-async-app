<objective>
Wire skill check resolution into the take-turn server action using a two-call architecture. Call #1 extends the existing JSON schema with optional skillName; if valid, resolveSkillCheck runs, builds a compact MECHANICAL CONTEXT keyword, and Call #2 produces the narration with that context. SkillCheckCard renders in the game UI showing the skill outcome in violet (superseding DiceCard on skill check turns).

Purpose: Fulfills SKILL-04 — code owns all dice math; Claude receives only the outcome keyword, not raw numbers. Maintains the rule that Claude cannot alter mechanical values.

Output: Updated take-turn.ts with two-call architecture; updated page.tsx with SkillCheckCard component and state management.
</objective>
