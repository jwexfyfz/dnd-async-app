<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → takeTurn(chipText) | User-supplied text crosses into server action |
| Claude response → stateDeltas | AI-generated JSON crosses into game state mutation |
| concurrent requests → $transaction | Two simultaneous submissions race to mutate game state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | chipText prompt injection | mitigate | sanitizeChipText strips SYSTEM:/ASSISTANT:/USER: prefixes, backticks, "ignore previous", enforces 200-char max before any use |
| T-04-02 | Tampering | stateDeltas hp/xp/level from Claude | mitigate | Explicit delete of hp, maxHp, xp, level, proficiencyBonus from stateDeltas before apply; rules engine owns these fields exclusively |
| T-04-03 | Tampering | concurrent turn submission race | mitigate | prisma.$transaction with version optimistic lock; concurrent write throws STALE_TURN and rolls back |
| T-04-04 | Tampering | Claude altering dice roll outcome | mitigate | rollD20Check called before narration call; D20Result is passed as a FACT to Claude; system prompt forbids Claude from changing roll outcomes |
| T-04-05 | Denial of Service | chipText exhausting Claude token budget | mitigate | 200-char max on sanitizedAction limits token consumption from user input |
| T-04-06 | Information Disclosure | consecutiveMisses counter in system prompt | accept | Counter is game-internal state visible to the DM; no PII; low value |
</threat_model>
