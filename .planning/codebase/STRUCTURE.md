# Codebase Structure

**Analysis Date:** 2026-05-20

## Directory Layout

```
dnd-async-app/
├── app/                  — Next.js App Router: pages, layouts, routes, server actions
│   ├── actions/          — Server Actions ("use server") — all backend data logic
│   ├── api/              — Route Handlers (currently empty aside from placeholder auth dir)
│   │   └── auth/         — (empty) placeholder for future NextAuth or webhook routes
│   ├── auth/
│   │   └── callback/     — OAuth redirect landing (route.ts redirects to /)
│   ├── globals.css       — Global TailwindCSS styles
│   ├── layout.tsx        — Root layout: fonts, html/body wrapper, metadata
│   └── page.tsx          — Home page: auth gate, character form, character roster
│
├── components/           — Shared React client components
│   ├── character-form.tsx  — Point-buy character creation form
│   ├── character-list.tsx  — Character roster display grid
│   └── login-screen.tsx    — Google OAuth sign-in button
│
├── lib/                  — Shared utility modules (server and client)
│   ├── prisma.ts         — Prisma singleton with Neon serverless adapter
│   ├── supabase-client.ts — Browser-side Supabase client (for auth state)
│   └── supabase-server.ts — Server-side Supabase client factory (per-request, cookie-based)
│
├── prisma/               — Database schema and seed scripts
│   ├── schema.prisma     — Prisma schema: User, Character, Map, StoryPrompt, Game, Message
│   └── seed.mjs          — Seed script: 2 maps + 2 story prompts
│
├── generated/            — Auto-generated Prisma client (committed, regenerated by postinstall)
│   └── prisma/
│       └── client/       — Generated type definitions and runtime
│
├── public/               — Static assets (SVGs only currently)
├── CLAUDE.md             — Project instructions and conventions for Claude
├── next.config.ts        — Next.js config: serverExternalPackages for pg/dotenv
├── prisma.config.ts      — Prisma CLI config: loads .env.local, uses DIRECT_URL
├── tsconfig.json         — TypeScript config
├── postcss.config.mjs    — PostCSS config for Tailwind v4
└── eslint.config.mjs     — ESLint config (Next.js preset)
```

## Key Files

**Entry Points:**
- `app/layout.tsx`: Root HTML shell, Geist font injection, global CSS.
- `app/page.tsx`: Only page route. Owns auth state and character roster state. Renders `LoginScreen` if unauthenticated, else `CharacterForm` + `CharacterList`.

**Auth:**
- `lib/supabase-client.ts`: Browser Supabase client singleton — used in `page.tsx` for `onAuthStateChange` and `setSession`.
- `lib/supabase-server.ts`: Per-request server Supabase client factory — called at the top of every server action to verify the session from cookies.
- `app/auth/callback/route.ts`: OAuth redirect receiver — redirects to `/` to let `page.tsx` process the hash token.

**Database:**
- `lib/prisma.ts`: Prisma singleton with `@prisma/adapter-neon`. Uses `globalThis` to avoid duplicate clients in dev hot-reload.
- `prisma/schema.prisma`: Canonical schema. Six models: `User`, `Character`, `Map`, `StoryPrompt`, `Game`, `Message`.
- `prisma/seed.mjs`: Idempotent seed (upsert by name). Populates 2 maps and 2 story prompts.
- `prisma.config.ts`: Tells Prisma CLI to use `DIRECT_URL` (not the pooled `DATABASE_URL`) for migrations.

**Server Actions:**
- `app/actions/create-character.ts`: Auth-gated. Upserts User, creates Character. Calls `revalidatePath("/")`.
- `app/actions/get-characters.ts`: Auth-gated. Fetches all characters for the current user, includes active game IDs.
- `app/actions/start-game.ts`: Auth-gated. Idempotent game creation. Computes starting HP, builds initial `Game.state` JSON.
- `app/actions/get-story-prompts.ts`: No auth required. Returns all seeded `StoryPrompt` rows ordered by difficulty.

**Components:**
- `components/login-screen.tsx`: Google OAuth button. Constructs Supabase OAuth URL dynamically using `window.location.origin` (works in both local dev and production).
- `components/character-form.tsx`: Full point-buy stat allocation UI with inline D&D 5e rule enforcement and user-facing hint messages.
- `components/character-list.tsx`: Pure display component. Receives characters as props, renders stat grid cards.

## Module Organization

Features are not separated into feature folders. Organization is by **layer type**:

| Layer | Location | Role |
|-------|----------|------|
| Pages / Routes | `app/` | UI entry points and route handlers |
| Server Actions | `app/actions/` | All data reads and writes |
| UI Components | `components/` | Reusable React client components |
| Shared Utilities | `lib/` | DB client, auth client factories |
| Schema | `prisma/` | Data model and seed data |

This is a flat, layer-based structure rather than domain/feature-based. All character-related code spans `app/actions/create-character.ts`, `app/actions/get-characters.ts`, `components/character-form.tsx`, and `components/character-list.tsx` — no enclosing `character/` feature folder.

## Naming Conventions

**Files:**
- Pages: `page.tsx`, `layout.tsx` (Next.js App Router convention)
- Route Handlers: `route.ts`
- Server Actions: `kebab-case.ts` describing the action (e.g., `create-character.ts`, `get-story-prompts.ts`)
- Components: `kebab-case.tsx` (e.g., `character-form.tsx`, `login-screen.tsx`)
- Lib utilities: `kebab-case.ts` with scope suffix (e.g., `supabase-client.ts`, `supabase-server.ts`)

**Directories:**
- `app/actions/`: Flat — no subdirectories by domain yet
- `components/`: Flat — no subdirectories by feature yet
- `lib/`: Flat — shared utilities only

**Components:**
- Named exports only for lib utilities (`export const prisma`, `export async function createSupabaseServerClient`)
- Default exports for React components and page files

**Interfaces:**
- Defined locally within the file that uses them (no shared `types/` directory)
- `Character` interface duplicated between `page.tsx` and `character-list.tsx` — not yet centralized

## Where to Add New Code

**New server action (data read/write):**
- Create `app/actions/<verb>-<noun>.ts`
- Add `"use server"` directive at top
- First line of any user-data action: `const supabase = await createSupabaseServerClient()` + auth check

**New page/route:**
- Create `app/<route-name>/page.tsx` for a new page
- Create `app/api/<route-name>/route.ts` for a new API route handler

**New UI component:**
- Create `components/<component-name>.tsx`
- Add `"use client"` if it uses React hooks or browser APIs
- Pass data as props from the page; do not fetch data directly inside components

**New database model:**
- Edit `prisma/schema.prisma`
- Run `npx prisma migrate dev` (dev) or check CLAUDE.md constraints before running migrations
- Add seed data to `prisma/seed.mjs` if it's static content

**Shared types:**
- No `types/` directory exists yet. Currently defined inline per file.
- To centralize: create `lib/types.ts` and import from there.

## Special Directories

**`generated/`:**
- Purpose: Auto-generated Prisma client types and runtime
- Generated: Yes — by `prisma generate` (run automatically via `postinstall` npm hook)
- Committed: Yes (to ensure Vercel build has access without running generate separately)

**`.planning/`:**
- Purpose: GSD planning documents (codebase maps, phase plans)
- Generated: By Claude planning agents
- Committed: Yes

**`.claude/`:**
- Purpose: Claude agent skills and configuration
- Generated: No
